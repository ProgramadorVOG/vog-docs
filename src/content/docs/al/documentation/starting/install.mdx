Es esencial que, antes de comenzar cualquier proyecto en AL (Application Language) para Microsoft Dynamics 365 Business Central, la preparación del ambiente de desarrollo es un paso crucial. Esta preparación típicamente incluye la instalación y configuración de las siguientes herramientas y extensiones:

**1. Visual Studio Code (VS Code)**

VS Code es el editor de código principal y la plataforma recomendada para el desarrollo en AL. Es ligero, potente y admite una gran cantidad de extensiones necesarias.


- **Recomendación:** Asegúrate de tener la última versión estable instalada.

<LinkCard
  title="Visual Studio Code (Editor de Codigo)"
  href="https://code.visualstudio.com/"
  target="_blank"
/>

**2. Git**

Git es el sistema de control de versiones estándar de la industria. Es fundamental para cualquier desarrollo colaborativo y para gestionar las versiones del código de forma segura.

    **Configuración:** Debe estar instalado en tu sistema y, preferiblemente, configurado con una cuenta en un servicio de repositorio GitHub.

<LinkCard
  title="Pagina oficial Git"
  href="https://git-scm.com/downloads"
  target="_blank"
/>

**3. Extensión de Microsoft Dynamics 365 (AL Language)**

Esta es la extensión oficial y obligatoria de Microsoft para el desarrollo en AL dentro de VS Code.

    **Nombre Oficial en el Marketplace:** AL Language (por Microsoft).

<LinkCard
  title="AL Language extension for Microsoft Dynamics 365 Business Central"
  href="https://marketplace.visualstudio.com/items?itemName=ms-dynamics-smb.al"
  target="_blank"
/>

**4. Extensión AZ AL Dev Tools/AL Code Outline**

Esta extensión, creada por Andrzej Zwierzchowski, es altamente recomendada para mejorar la productividad en el desarrollo AL. Permite visual los objetos nativos de Microsoft Dynamics Business Central

<LinkCard
  title="AZ AL Dev Tools/AL Code Outline"
