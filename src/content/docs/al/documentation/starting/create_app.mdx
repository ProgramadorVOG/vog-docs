La publicación de un archivo **.app** de extensión en un entorno de Producción de Business Central se realiza a través de la interfaz de usuario del Centro de Administración (Admin Center) o mediante la administrador de extensiones, 

<Aside type="caution">
 las instancias en la nube no permiten la publicación directa desde Visual Studio Code, que está reservado para entornos de desarrollo y Sandbox.
</Aside>

## Crear el Archivo XML de Permisos

El conjunto de permisos define a qué objetos de tu extensión (y objetos base de Business Central si es necesario) pueden acceder los usuarios.

En tu proyecto de VS Code, Ejecuta en la paleta de comandos el comando **AL:Generate Permission set as XML file containing current extension**. El comando crear un archivo xml con los permisos establecidos para todos los objecto creados en tu proyecto.


<Aside type="caution">
  Si se actualiza una extension actualmente en produccion con nuevos objetos, se debe eliminar el archivo xml y volver a crear.

</Aside>

## Generar el Archivo .app para Producción


Una vez que el archivo xml de permsisos está listo, el conjunto de permisos se compilará dentro de tu archivo `.app` al ejecutar el comando  al presionar `F5`.

<Aside>
    Asegúrate de que tu código está finalizado y probado en un entorno Sandbox
</Aside>

## Instalar la Extensión desde la Interfaz de Usuario de Business Central

Abre la instancia de Producción de Business Central y sigue los siguientes pasos:

- Usa el buscador (la lupa) para ir a la página de Administrador de Extensiones (Extension Management).

- Busca la opcion de **Instalar extensión**.

- Sigue el asistente de instalación.

Una vez instalada, la extensión estará activa y disponible para todos los usuarios en el entorno de Producción.

<Aside type='caution'>
    Asegúrate de que la extensión se ha instalado, buscando en la lupa de Business Central **Estado de instalación de extension** y verificando el estado completado en la extensión instalada.
</Aside>

## Consideración Importante: Versiones y Actualizaciones

Si ya existe una versión anterior de tu extensión instalada, debes incrementar el número de versión en el archivo `app.json` antes de generar el nuevo archivo .app y cargarlo.

Al cargar una versión más nueva, el sistema ejecutará un proceso de actualización (Upgrade) de la extensión para migrar los datos si es necesario. Asegúrate de que tus objetos AL contienen código de actualización si has realizado cambios destructivos.
