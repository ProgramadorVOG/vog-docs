### Fechas

La codificación de **la fecha** debe seguir un formato específico (ya no es específico de la región). Use el formato aaaammddD. Por ejemplo, 20170825D.

### Servicios HTTP y HTTPS

Los servicios remotos (incluidas todas las llamadas a servicios web) pueden usar HTTP o HTTPS. Sin embargo, las llamadas HTTP solo son posibles mediante el tipo AL HttpRequest. Consulte la [descripción general de HTTP](https://learn.microsoft.com/en-us/dynamics365/business-central/dev-itpro/developer/devenv-restapi-overview/) para obtener más información sobre este tema.

### Control Addin

Solo se admiten complementos de cliente web basados en JavaScript. El compilador gestiona automáticamente el proceso de compresión. Simplemente incluya el nuevo tipo de complemento de control AL, las fuentes JavaScript y compile la aplicación. Consulte " Objeto de complemento de control" para obtener más información sobre este tema.

### Firma Digital

El archivo .app debe estar firmado digitalmente. Consulte [Firmar un archivo de paquete APP](https://learn.microsoft.com/en-us/dynamics365/business-central/dev-itpro/developer/devenv-sign-extension/) para obtener más información sobre este tema.

### Documentación de escenario

El documento del escenario de usuario debe contener los pasos detallados para todas las pruebas de configuración y validación del usuario. Consulte [la documentación del escenario de usuario](https://learn.microsoft.com/en-us/dynamics365/business-central/dev-itpro/compliance/apptest-userscenario/) para obtener más información sobre este tema

### Aplicacion de area en los controles

Defina las áreas de aplicación aplicables a sus controles. Si no lo hace, el control no aparecerá en Dynamics 365 Business Central. Consulte [la propiedad Área de aplicación](https://learn.microsoft.com/en-us/dynamics365/business-central/dev-itpro/developer/properties/devenv-applicationarea-property/) para obtener más información sobre este tema.

### Permisos

Los conjuntos de permisos deben ser creados por la extensión y, al estar marcados, otorgar al usuario todas las funciones de configuración y uso. No se requiere que el usuario tenga permisos SUPER para configurar y usar la extensión. Consulte la siguiente información sobre este tema.

### AddFirst y AddLast

Usa " `addfirst`y `addlast`" para colocar tus acciones en las páginas de Business Central. Esto evita que la aplicación se dañe debido a cambios en el núcleo de Business Central. Consulta [Colocar acciones y controles](https://learn.microsoft.com/en-us/dynamics365/business-central/dev-itpro/developer/devenv-page-ext-object#using-keywords-to-place-actions-and-controls) para obtener más información sobre este tema.

### Valor ToBeClassified

• La clasificación de datos es obligatoria para los campos de todas las extensiones de tabla. La propiedad debe tener un valor distinto de "ToBeClassified". Consulte ["Clasificación de datos"](https://learn.microsoft.com/en-us/dynamics365/business-central/dev-itpro/developer/devenv-classifying-data/) para obtener más información sobre este tema.

### OnBeforeCompanyOpen y OnAfterCompanyOpen

• No utilice OnBeforeCompanyOpen ni OnAfterCompanyOpen. Consulte ["Sustitución de](https://learn.microsoft.com/en-us/dynamics365/business-central/dev-itpro/compliance/apptest-onbeforecompanyopen/) estos eventos" para obtener más información sobre este tema.

### Objetos Perfiles

Debe usar el objeto Perfil para agregar perfiles en lugar de insertarlos en la tabla Perfiles. Consulte [Objeto Perfil](https://learn.microsoft.com/en-us/dynamics365/business-central/dev-itpro/developer/devenv-profile-object/) para obtener más información sobre este tema.

### Prefijos y Sufijos

Debe añadir un prefijo o sufijo al nombre de sus campos y objetos. Esto evita conflictos entre aplicaciones. Consulte la [sección "Beneficios y directrices"](https://learn.microsoft.com/en-us/dynamics365/business-central/dev-itpro/compliance/apptest-prefix-suffix/) para obtener más información sobre este tema.

### Servicios Web

Las páginas y unidades de código diseñadas para exponerse como servicios web no deben generar ninguna interfaz de usuario que provoque una excepción en el código de llamada. Consulte [Restricciones de la interfaz de usuario para objetos expuestos como servicios web](https://learn.microsoft.com/en-us/dynamics365/business-central/dev-itpro/compliance/apptest-webservices/) para obtener más información sobre este tema.

### Codigo de Actualizacion

Incluye el código de actualización adecuado para que tu aplicación se actualice correctamente entre versiones. Consulta ["Actualizar extensiones"](https://learn.microsoft.com/en-us/dynamics365/business-central/dev-itpro/developer/devenv-upgrading-extensions/) para obtener más información sobre este tema.

### Probar Exhaustivamente

Pruebe exhaustivamente su extensión en un entorno de Dynamics 365 Business Central. Consulte ["Probar su extensión"](https://learn.microsoft.com/en-us/dynamics365/business-central/dev-itpro/compliance/apptest-testingyourextension/) para obtener más información sobre este tema.

Antes de enviar la aplicación para validación, asegúrese de poder publicar, sincronizar, instalar, desinstalar y reinstalar su extensión. Esto debe hacerse en un entorno de Dynamics 365 Business Central. Consulte " [Cómo publicar su aplicación"](https://learn.microsoft.com/en-us/dynamics365/business-central/dev-itpro/developer/devenv-how-publish-and-install-an-extension-v2/) para obtener más información sobre este tema.

### Paquetes de Pruebas

Debe incluir un paquete de pruebas de Visual Studio Code con su extensión. Asegúrese de incluir la mayor cobertura de código posible. Consulte " [Probar la extensión de ejemplo avanzada"](https://learn.microsoft.com/en-us/dynamics365/business-central/dev-itpro/developer/devenv-extension-advanced-example-test/) para obtener más información sobre este tema.
